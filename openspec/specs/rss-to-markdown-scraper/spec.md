## Purpose

Define the RSS-to-markdown scraper CLI behavior, metadata, and compatibility requirements.

## Requirements

### Requirement: RSS feed ingestion
The CLI SHALL accept an RSS feed URL and parse the feed into individual posts. The CLI MUST also support processing RSS input from repository-local fixtures so that the workflow can be executed without network access.

#### Scenario: Parse feed into posts
- **WHEN** the user runs the CLI with a valid RSS feed URL
- **THEN** the tool parses the feed and identifies each post item for export

#### Scenario: Parse repository-local feed fixture
- **WHEN** the CLI is executed against a repository-local RSS fixture
- **THEN** the tool parses the feed and identifies each post item for export without network access

### Requirement: Markdown export with YAML frontmatter
Each parsed post SHALL be written to a markdown file that includes YAML frontmatter metadata.

#### Scenario: Write post to markdown
- **WHEN** a feed item is parsed
- **THEN** the tool writes a markdown file with YAML frontmatter and the post body content

### Requirement: Required frontmatter fields
The YAML frontmatter SHALL include `title`, `date`, `link`, and `guid` fields for each post.

#### Scenario: Populate required metadata
- **WHEN** a feed item provides title, publication date, link, and guid
- **THEN** the generated frontmatter includes those values in `title`, `date`, `link`, and `guid`

### Requirement: Blogger feed support
The CLI SHALL successfully parse and export RSS feeds generated by Blogger.

#### Scenario: Parse Blogger RSS feed
- **WHEN** the CLI ingests a Blogger RSS feed
- **THEN** the tool exports each post to markdown with frontmatter and body content

### Requirement: WordPress feed support
The CLI SHALL successfully parse and export RSS feeds generated by WordPress.

#### Scenario: Parse WordPress RSS feed
- **WHEN** the CLI ingests a WordPress RSS feed
- **THEN** the tool exports each post to markdown with frontmatter and body content
