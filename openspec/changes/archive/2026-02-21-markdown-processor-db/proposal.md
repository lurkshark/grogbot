## Why

We need a local, offline-friendly way to embed scraped markdown posts so downstream tooling can search and analyze content without relying on external vector services. This enables faster iteration on post-processing workflows and keeps data close to the filesystem produced by `@grogbot/goblin`.

## What Changes

- Introduce a new CLI package `@grogbot/alchemist` that accepts a directory of markdown files generated by `@grogbot/goblin`.
- For each markdown file, generate embeddings from the file content using the configured embedding model.
- Store the embedding vector and associated post content in a local-file vector database.
- Provide a repeatable ingestion workflow (batch run) for populating the local vector store from the input directory.

## Capabilities

### New Capabilities
- `markdown-processor-db`: Ingest markdown files, embed content, and persist vectors plus content to a local-file vector database via a CLI.

### Modified Capabilities
- (none)

## Impact

- New package under the monorepo: `@grogbot/alchemist` (CLI entrypoint, embeddings integration, local vector DB storage).
- Dependencies on an embedding model/provider and a local-file vector database library or implementation.
- New runtime behavior for batch processing of markdown files from `@grogbot/goblin` output directories.
